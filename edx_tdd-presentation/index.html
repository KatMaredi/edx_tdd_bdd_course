1. Test Coverage

What is test coverage?

The percentage of lines of code executed during tests.

What is the importance of test coverage?

  • The higher the test coverage, the more confident you get in your code.
  • Test coverage reports show which lines of code were not tested

2. Factories and Fakes.

Factories

Factories are classes/objects responsible for creating instances of an object.

Fakes

Fakes are the fake data generated by the factory that mimics real data

Why are they important?

  • Sometimes we need fake data to test against, we use them to generate fake data for testing.
  • Sometimes we also need a fake class with fake data to test.
